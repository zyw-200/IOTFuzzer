IP Settings

This page lets you to configure your Access Point as a client of a Dynamic Host Configuration Protocol (DHCP) server. It has the following options:

    • DHCP Client: Enable this option if you want the Access Point to get its IP address and other network settings from a DHCP server on your LAN. By default, the DHCP client option is disabled. If you enable it, the IP address, subnet mask, default gateway, primary DNS, and Secondary DNS settings fields are disabled, because the Access Point automatically gets them from the DHCP server on your network.
      If you enable this option, you can also access the Access Point's administrative screen by entering the Access Point name in the URL field of the browser.
    • IP Address: This is the IP address of the Access Point. The default IP address is 192.168.0.237. To change it, enter an unused IP address from the address range used on your LAN.
    • Subnet Mask: Enter the subnet mask value used on your LAN. The default value is 255.255.255.0.
    • Default Gateway: Enter the IP address of the gateway for your LAN.
    • Primary DNS Server: Enter the IP address of the Primary Domain Name Server (DNS) that you want to use.
    • Secondary DNS Server: Enter the IP address of the Secondary Domain Name Server (DNS) that you want to use.
    • Network Integrity Check: Check this box to enable the WNAP320 to validate that the upstream link is active before allowing wireless associations. If you set this option you must ensure your default gateway is configured.

802.1Q VLAN

The 802.1Q VLAN protocol on the Access Point logically separates traffic on the same physical network.

    • Untagged VLAN

      When checked, this option allows one VLAN to be configured as an "untagged VLAN". When the Access Point sends frames associated with the untagged VLAN out the LAN (Ethernet) interface, those frames will be untagged. When the Access Point receives untagged traffic from the LAN (Ethernet) interface, those frames are assigned to the untagged VLAN.

      If unchecked, the Access Point tags all outgoing LAN (Ethernet) frames, and only accepts incoming frames that are tagged with known VLAN IDs.

      Note: The Untagged VLAN checkbox should only be unchecked if the hubs or switches on your LAN support the VLAN (802.1Q) standard. Likewise, the Untagged VLAN value should only be changed if the hubs/switches on your LAN support the VLAN (802.1Q) standard. Changing either of these values will result in a loss of IP connectivity if the hubs/switches on your network have not yet been configured with the corresponding VLANs.

    • Management VLAN

      Management VLANs are used for managing traffic (Telnet, SNMP, and HTTP) to and from the Access Point.

      Frames belonging to the Management VLAN are not given any 802.1Q header when sent over the trunk. If a port is in a single VLAN, it can be untagged. But if the port needs to be a member of multiple VLANs, it must be tagged.
