Basic Edit Profile Settings Help

Edit Security Profile

This page lets you change the Security Profile settings of the profile that you selected on the Profile Settings page.

Profile Definition

Specify the following information in this section:

    • Profile Name: Enter a unique name to identify the Security Profile. This value can be up to 32 characters. Spaces are allowed, except at leading and trailing place which will be automatically removed. Use meaningful profile names instead of the default names. The default profile names are Profile1, Profile2, ... Profile8.
    • Wireless Network Name (SSID): Enter the name of your wireless network associated with this profile.
    • Broadcast Wireless Network Name (SSID): Enabled by default. If set to Yes, the SSID is broadcast. If you disable broadcast of the SSID, only devices that have the correct SSID can connect to the Access Point.

Authentication Settings

Specify the following information in this section:

    • Network Authentication: Enter the authentication type to be used. The following table summarizes the value and meaning for the available authentication types.

Type

Description

Open System

Can be used with WEP encryption or no encryption.

Shared Key

You must use WEP encryption and enter at least one shared key.

Legacy 802.1x

You must configure the RADIUS Server Settings to use this option.

WPA with RADIUS

You must configure the RADIUS Server Settings to use this option.

WPA2 with RADIUS

Only select this if all clients support WPA2. If selected, you must use AES encryption and configure the RADIUS Server Settings.

WPA & WPA2 with RADIUS

This selection allows clients to use either WPA (with TKIP) or WPA2 (with AES). If selected, you must use TKIP + AES encryption and configure the RADIUS Server Settings.

WPA-PSK

You must use TKIP or TKIP + AES encryption and enter the WPA pass-phrase (Network key).

WPA2-PSK

Only select this if all clients support WPA2. If selected, you must use AES and TKIP + AES encryption and enter the WPA pass phrase (Network key).

WPA-PSK and WPA2-PSK

This selection allows clients to use either WPA (with TKIP) or WPA2 (with AES). If selected, you must use TKIP + AES encryption and enter the WPA passphrase (Network key).

    • Data Encryption: Enter the data encryption type to be used. The options available for data encryption depend on the Network Authentication settings.

Data Encryption Type

Description

None

No encryption is used.

64 bits WEP

Standard WEP encryption, using 40/64 bit encryption.

128 bits WEP

Standard WEP encryption, using 104/128 bit encryption.

152 bits WEP

Proprietary mode that will only work with other wireless devices that support this mode.

TKIP

This is the standard encryption method used with WPA and WPA2.

AES

This is the standard encryption method for WPA2. Some clients may support AES with WPA, but this is not supported by this Access Point.

TKIP + AES

This setting supports both WPA and WPA2. Broadcast packets use TKIP. For unicast (point-to-point) transmissions, WPA clients use TKIP, and WPA2 clients use AES.
    • Show Passphrase in Clear Text : Selecting Yes will allow user to view the Passphrase in Characters. Default always remains as No, which does not displays Passphrase in Characters.
    • Wireless Client Security Separation: Specify if you want the associated wireless clients to be able to communicate with each other. If enabled, the associated wireless clients will not be able to communicate with each other. This feature is intended for hotspots and other public access situations.
    • Dynamic VLAN: This option is used to configure Dynamic/Static VLAN feature. When option Disable is selected, it implies that Static VLAN feature is enabled and user can configure Static VLAN parameters. This is also the default option for this field. When option Optional is selected, it implies that Dynamic VLAN feature is activated but even if Radius Server doesnot return any VLAN-ID, still the station will be allowed to connect to the Access Point. When option Required is selected, it implies that the Dynamic VLAN feature is activated and it is mandatory to get a VLAN-ID from Radius Server, failing to which the station will be de-authenticated. Dynamic VLAN featyure can be enabled only when Authentication is set to any Radius type authentication. This includes the Security Profile's Network Authentication or Remote MAC Address Database authentication for MAC Authentication feature.
    • VLAN ID: The default VLAN ID to be associated with this security profile. The VLAN ID must match the VLAN ID used by other network devices.
    • Access Control: This is enabled only for Static VLAN feature, that is when Dynamic VLAN is set to Disable. This specifies to enable or disable access control on the profile. Default value is Disable.
    • Access Control Policy: This is enabled only for Static VLAN feature, that is when Dynamic VLAN is set to Disable. This define whether to connect the STA to default configured VLAN or just send disconnect to the STA if the vlan id returned from the Radius server does not match with the one statically configured on the vap. Default value is Disable.
