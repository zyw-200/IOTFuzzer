Basic General Help

General

This page lets you configure the basic settings of your Access Point.

Access Point Name

This unique value indicates the Access Point NETBIOS name. The default Access Point Name is on the bottom label of the WNAP320. By default, the name is netgearxxxxxx, where xxxxxx represents the last six hexadecimal digits of the WNAP320 MAC address.

NETGEAR recommends changing the name as soon as possible after setting up the Access Point. The name must contain only alphabets, numbers and hyphens, and must be 15 characters or less.

Country/Region

This field displays the region of operation for which the Access Point is intended.

For products sold in the United States, the default country is preset and can not be changed. For products sold outside of the United States, you must select a country or region. It may not be legal to operate the access point in a country/region other than the country/region shown here. If your country or region is not listed, you must check with your local government agency or check the NETGEAR Website for more information on which channels to use.

Cloud Enabled

This field is the configuration parameter which gives the cloud agent running status. The options available are yes and no. If the cloud agent is currently running, then yes is selected otherwise no is selected. On the other hand if the user wants to stop the agent from running user needs to click on "no" and click on apply. If the user wants to run the cloud agent then the user needs to click on "yes" and apply. When the AP comes with factory defaults the agent was already running and the user can see "yes" already selected in the GUI.

When the agent is running, it is connected to the cloud online service, registers itself with the service and then onwards helps the online service (Cloud WMS) to configure the AP, monitor the AP and receive notifications from the AP.

Cloud Activation Status

There are two possible states defined for this value. This is a read only value in the GUI and displays the status of activation.

a) Pending - The activation cycle must be started or is underway. The cloud agent could be running or not running in this state. Usually when the AP comes up with factory defaults, this is the usual default state. The change of state is usually from pending to activated.

b) Activated - The cloud agent has successfully completed an activation cycle against the Cloud Online Service. An AP when added to the cloud using Serial Number to a location and when the agent communicates with the cloud and registers itself, then the AP gets activated.

Cloud Connectivity Status

There are two possible states and it represents the connection state of the agent with the cloud online service

a) Connected - Agent is connected to the Cloud WMS and is currently communicating actively. The AP should be able to access internet for this.

b) Disconnected - Agent is not connected to the Cloud WMS is not able to communicate. The disconnection could happen due to cable plug off, internet down, unreachable gateway, DNS resolution problems etc.

The Link to Cloud UI or Cloud WMS is wireless.netgear.com

The user can use this link to create and access the cloud WMS account and add the AP to the cloud if not already added. The AP can be managed using a cloud account and monitor it.
